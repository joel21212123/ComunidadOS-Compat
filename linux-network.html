Configuración de Red en Linux: Conectividad y Solución de Problemas

Configurar la red en Linux puede ser tan sencillo como conectarse a una Wi-Fi o tan complejo como configurar un servidor con múltiples interfaces. Afortunadamente, las distribuciones modernas de Linux han simplificado enormemente el proceso para el usuario promedio, mientras que aún ofrecen la flexibilidad y el control que los usuarios avanzados y administradores necesitan. Esta guía te ayudará a entender los conceptos básicos y a solucionar problemas comunes.

Conceptos Básicos de Red en Linux

  • Interfaz de Red: Es el hardware o software que permite a tu computadora conectarse a una red (ej., Ethernet, Wi-Fi, Bluetooth). Se identifican con nombres como `eth0`, `wlan0`, `enpXsY`, `wlXsY`.
  • Dirección IP: Un identificador único para tu dispositivo en la red (ej., `192.168.1.100`). Puede ser dinámica (DHCP) o estática.
  • Máscara de Subred: Define qué parte de la dirección IP es la red y qué parte es el host.
  • Puerta de Enlace (Gateway): La dirección IP del router que conecta tu red local a Internet.
  • Servidores DNS: Convierten nombres de dominio (ej., google.com) en direcciones IP.
  • NetworkManager: La herramienta gráfica más común para gestionar conexiones de red en distribuciones de escritorio.

Configuración de Red Gráfica (NetworkManager)

La forma más común y sencilla de gestionar tus conexiones en Ubuntu, Fedora, y la mayoría de los escritorios:

  1. Acceder a la Configuración de Red: Busca "Configuración" (o "Settings") y luego la sección "Red" (o "Network").
  2. Conectarse a Wi-Fi:
    • Selecciona la pestaña "Wi-Fi".
    • Activa el Wi-Fi si no lo está.
    • Selecciona tu red de la lista y haz clic en "Conectar" (o el botón con la rueda dentada para opciones avanzadas).
    • Introduce la contraseña si es necesario.
  3. Configurar una Conexión por Cable (Ethernet):
    • Conecta el cable Ethernet. Generalmente, la conexión se establece automáticamente por DHCP.
    • Para configurar IP estática, selecciona la conexión por cable, haz clic en la rueda dentada, ve a la pestaña "IPv4" o "IPv6", cambia a "Manual" y rellena los detalles.
  4. Opciones Avanzadas: Puedes configurar servidores DNS personalizados, proxies, y otras opciones en las propiedades de cada conexión.

Configuración de Red por Línea de Comandos (CLI)

Para usuarios avanzados o para servidores, la CLI es indispensable:

  • Verificar el Estado de la Red: 
    
ip a          # Muestra todas las interfaces de red y sus IPs
ip route      # Muestra la tabla de enrutamiento
ping google.com # Envía paquetes para verificar conectividad a un host
ss -tuln      # Lista los puertos abiertos y en uso
nmcli device show # (NetworkManager CLI) Muestra detalles de los dispositivos de red
  • Activar/Desactivar Interfaces: 
    
sudo ip link set eth0 up    # Activa la interfaz eth0
sudo ip link set eth0 down  # Desactiva la interfaz eth0
  • Configurar IP Estática (Ejemplo): 
    
# Esto es solo un ejemplo; la configuración permanente varía por distribución (netplan, network-scripts, systemd-networkd)
sudo ip addr add 192.168.1.100/24 dev eth0
sudo ip route add default via 192.168.1.1
  • Gestionar Conexiones Wi-Fi con `nmcli`: 
    
nmcli device wifi list        # Lista redes Wi-Fi disponibles
nmcli device wifi connect "NombreDeTuRed" password "TuContraseña" # Conecta a una red Wi-Fi

Solución de Problemas Comunes de Red

  • No hay conexión a Internet:
    • Verifica que el cable Ethernet esté conectado o que el Wi-Fi esté activado.
    • Reinicia tu router y tu PC.
    • Asegúrate de que estás conectado a la red correcta.
    • Abre el terminal y prueba `ping 8.8.8.8` (DNS de Google). Si funciona, el problema es DNS.
    • Si `ping 8.8.8.8` no funciona, prueba `ping 192.168.1.1` (o la IP de tu router). Si esto funciona, el problema es el gateway o la conexión a Internet.
    • Comprueba la configuración de IP (DHCP o estática).
    • Verifica los drivers de tu tarjeta de red.
  • Wi-Fi no detectado o lento:
    • Asegúrate de que el módulo Wi-Fi esté cargado (`lsmod | grep -i wifi`).
    • Instala drivers propietarios si es necesario (especialmente para Broadcom o Realtek).
    • Verifica interferencias o una señal débil.
  • DNS no funciona:
    • Revisa la configuración de DNS en tu NetworkManager o en `/etc/resolv.conf`.
    • Prueba con DNS públicos (ej., 8.8.8.8, 1.1.1.1).
  • Problemas de Firewall:
    • Asegúrate de que tu firewall (ufw, firewalld) no esté bloqueando las conexiones que necesitas.
    • Puedes deshabilitarlo temporalmente para probar (`sudo ufw disable` o `sudo systemctl stop firewalld`) y volver a activarlo después.

La red en Linux es un campo vasto. Empezar con la interfaz gráfica y luego explorar la línea de comandos te dará un control total sobre tu conectividad.
